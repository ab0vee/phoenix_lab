# Инструкция по получению API ключа Groq

Подробное руководство по настройке Groq API для Phoenix Lab.

## Шаг 1: Регистрация на Groq

1. Перейдите на сайт [Groq Console](https://console.groq.com/)
2. Нажмите кнопку **"Sign Up"** или **"Sign In"**
3. Войдите через Google, GitHub или создайте аккаунт с email
4. Подтвердите email, если требуется

## Шаг 2: Создание API ключа

1. После входа в консоль Groq перейдите в раздел **"API Keys"** (в меню слева или в настройках)
2. Нажмите кнопку **"Create API Key"** или **"Generate New Key"**
3. Заполните форму:
   - **Name** (необязательно): например, "Phoenix Lab"
   - Выберите права доступа (обычно достаточно базовых прав)
4. Нажмите **"Create"** или **"Generate"**
5. **ВАЖНО:** Скопируйте API ключ сразу - он показывается только один раз!
   - Формат: `gsk_xxxxxxxxxxxxxxxxxxxxxxxxxxxxx` (длинная строка, начинается с `gsk_`)
   - Сохраните ключ в безопасном месте

## Шаг 3: Выбор модели (опционально)

Groq предоставляет несколько моделей на выбор:

- **llama-3.1-70b-versatile** (по умолчанию) - универсальная модель, хороший баланс
- **llama-3.1-8b-instant** - быстрая модель для коротких текстов
- **mixtral-8x7b-32768** - модель с большим контекстом
- **gemma-7b-it** - модель от Google

Вы можете указать модель в `.env` файле через переменную `GROQ_MODEL`.

## Шаг 4: Добавление ключа в проект

1. Откройте файл `.env` в корне проекта (или создайте его, если нет)
2. Добавьте следующие строки:
   ```
   GROQ_API_KEY=gsk_xxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   GROQ_MODEL=llama-3.1-70b-versatile
   ```
3. Замените значение на ваш реальный API ключ:
   - `GROQ_API_KEY` - API ключ из шага 2
   - `GROQ_MODEL` - (опционально) модель для использования

## Шаг 5: Проверка настройки

1. Запустите backend сервер:
   ```bash
   cd Backend
   python server.py
   ```
2. Проверьте логи - должно быть сообщение:
   ```
   INFO: Groq API настроен
   INFO: Groq модель: llama-3.1-70b-versatile
   ```
3. Если видите предупреждение о том, что API не настроен, проверьте:
   - Правильность ключа в `.env`
   - Отсутствие пробелов вокруг знака `=`
   - Правильность названия переменной `GROQ_API_KEY`
   - Перезапустите backend сервер после изменения `.env`

## Важные замечания

- **Бесплатный тариф:** Groq предоставляет бесплатный тариф с лимитами на количество запросов
- **Скорость:** Groq известен очень высокой скоростью обработки запросов
- **Лимиты:** Проверьте текущие лимиты в консоли Groq
- **Безопасность:** Никогда не публикуйте API ключи в открытом доступе (GitHub, форумы и т.д.)

## Решение проблем

### Ошибка "Groq API не настроен"
- Проверьте, что ключ добавлен в `.env` файл
- Убедитесь, что нет пробелов вокруг `=`
- Перезапустите backend сервер после изменения `.env`

### Ошибка "Unauthorized" или "401"
- Проверьте правильность API ключа
- Убедитесь, что ключ начинается с `gsk_`
- Проверьте, что ключ не истёк (создайте новый, если нужно)

### Ошибка "Rate limit exceeded"
- Вы превысили лимит запросов
- Подождите некоторое время или проверьте лимиты в консоли Groq
- Рассмотрите возможность перехода на платный тариф

### Ошибка "Model not found"
- Проверьте правильность названия модели в `GROQ_MODEL`
- Убедитесь, что модель доступна в вашем регионе
- Попробуйте использовать модель по умолчанию: `llama-3.1-70b-versatile`
